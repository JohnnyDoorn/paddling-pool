Contingency Tables

Basics

When both your variables are categorical, the appropriate tools come from the chi-square family. A contingency table (cross-tabulation) shows the frequency of every combination of categories, and a chi-square test asks whether the two variables are independent.

This page covers:

  • Chi-square test of independence — are two categorical variables related?
  • Chi-square goodness-of-fit — does a single categorical variable match an expected distribution?
  • Fisher’s exact test — the small-sample alternative

Chi-square test of independence

What is it?

The chi-square test of independence asks: are two categorical variables related, or independent? It compares the observed cell frequencies in a contingency table against the expected frequencies under the null hypothesis of independence.

\[\chi^2 = \sum \frac{(O - E)^2}{E}\]

Example: do voting preferences (Party A / Party B / Party C) differ across age groups (18–30, 31–50, 51+)? The contingency table is 3 × 3; the null hypothesis is that age group and party preference are independent.

When to use it

  • Both variables are categorical (nominal or ordinal treated as nominal)
  • Each participant contributes to exactly one cell of the table (one row per participant)
  • You want to test whether membership in one variable’s categories is related to membership in the other’s

Assumptions

  • Independence: each observation is independent of the others (no participant appears in more than one cell)
  • Expected cell counts: all expected counts should be ≥ 5 (a common rule of thumb); at minimum, no more than 20% of cells should have expected counts below 5
  • If expected counts are too small, use Fisher’s exact test instead (see below)

Effect size

Report Cramér’s V (or φ for a 2 × 2 table):

Cramér’s V Conventional label (df = 1)
.10 Small
.30 Medium
.50 Large

For larger tables, the benchmarks scale down — consult DSUJ Chapter 19 or use JASP’s effect size output.

How to do it in JASP

Go to Frequencies → Contingency Tables in the analysis menu.

Interpreting the output

Statistic What it means
χ² The test statistic — bigger means observed frequencies deviate more from expected
df (rows − 1) × (columns − 1)
p Reject H₀ (independence) if p < α
Cramér’s V Effect size — the strength of association

If the test is significant, inspect the standardised residuals to see which cells contribute most to the deviation from independence.

TipAPA reporting example

A chi-square test of independence showed a significant association between age group and party preference, χ²(4) = 18.6, p < .001, Cramér’s V = .24. Inspection of the standardised residuals indicated that voters aged 51+ were over-represented among Party A supporters.

Chi-square goodness-of-fit

A variant of the chi-square test for one categorical variable against a hypothesised distribution. Example: does a die roll uniformly (each face expected at 1/6) over 600 throws?

In JASP: Frequencies → Multinomial Test. Specify either equal expected probabilities or a custom expected distribution.

Fisher’s exact test

When expected cell counts are too small for the chi-square approximation to be trustworthy (typically a 2 × 2 table with any expected count < 5), Fisher’s exact test computes the exact probability of the observed table under the null. It is available in the same JASP dialog — just tick Fisher’s exact test under Statistics.

TipWhy Fisher’s exact, not chi-square, with small expected counts

The chi-square statistic only approximately follows the χ² distribution; the approximation breaks down when expected counts are small. Fisher’s exact test side-steps the approximation altogether by enumerating every possible table with the same row and column totals. It costs nothing in interpretation — you still report a p-value — and is the safer choice for sparse tables.
