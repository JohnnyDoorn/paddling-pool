Logistic Regression

Modified

11 June 2026

Advanced Topics

Use logistic regression when your outcome variable is categorical — most commonly binary (yes/no, clinical/control, recovered/not recovered). Linear regression is inappropriate for binary outcomes because it can produce predicted probabilities below 0 or above 1, and it violates homoscedasticity.

What is it?

Binary logistic regression models the log-odds (logit) of the outcome as a linear function of the predictors:

\[\log\!\left(\frac{p}{1-p}\right) = b_0 + b_1 X_1 + b_2 X_2 + \ldots\]

The model predicts the probability that a case belongs to the “outcome = 1” category. Predictors can be continuous, binary, or categorical (with dummy coding).

Odds ratios

The exponentiated coefficients (e^b) are odds ratios (OR):

OR Interpretation
> 1 A one-unit increase in X is associated with increased odds of the outcome
< 1 A one-unit increase in X is associated with decreased odds of the outcome
= 1 No association
NoteOdds ≠ probability

An OR of 2 does not mean the outcome is “twice as likely” — it means the odds are doubled. Probabilities and odds are related but not the same. Always report predicted probabilities or absolute risk when communicating results to a non-statistical audience.

When to use logistic regression

Situation Appropriate method
Binary outcome (2 categories) Binary logistic regression
Nominal outcome (3+ unordered categories) Multinomial logistic regression
Ordinal outcome (3+ ordered categories) Ordinal logistic regression

This page focuses on binary logistic regression — the most common case in psychology research.

Assumptions

  • Binary outcome: the DV must have exactly two categories
  • Independence of observations: each participant contributes one row
  • No multicollinearity: predictors should not be highly correlated — check VIF as in regular regression
  • Sufficient cell sizes: especially important when predictors are categorical; very small cells can cause convergence problems
  • No complete separation: if a predictor perfectly predicts the outcome, the model cannot estimate coefficients
  • Linearity of continuous predictors with the log-odds: check using Box-Tidwell test or visual inspection (available as an option in JASP)

Note: logistic regression does not require normality of residuals or homoscedasticity — these are linear regression assumptions that do not apply here.

Model fit

Unlike linear regression, logistic regression uses maximum likelihood estimation. Model fit statistics:

Statistic What it tells you
−2 Log Likelihood (−2LL) Overall fit — smaller is better
Nagelkerke R² Pseudo-R²: approximate proportion of variance explained — treat as an effect size, not a literal R²
Hosmer-Lemeshow test Goodness of fit — a non-significant result indicates adequate fit
Classification table Percentage of cases correctly classified by the model

Model comparison: test whether adding predictors improves fit using the likelihood ratio test (χ² difference in −2LL with corresponding df).

Interpreting the output

Omnibus test: tests whether the model with predictors fits better than an intercept-only model. A significant χ² (p < .05) indicates the model explains a significant amount of variance.

Coefficients table: for each predictor, report: - b: the log-odds coefficient (hard to interpret directly) - OR (= e^b): the odds ratio — more interpretable - 95% CI for the OR: does it include 1.0? If not, the predictor is significant - p-value

TipAPA reporting example

Logistic regression was used to predict survival from age and treatment group. The model was significant, χ²(2) = 14.3, p < .001, Nagelkerke R² = .23. Treatment group was a significant predictor, b = 1.42, OR = 4.14, 95% CI [1.87, 9.17], p = .001.

Multinomial logistic regression

When the outcome has three or more unordered categories, use multinomial logistic regression. This fits separate binary logistic regressions comparing each category to a reference category, but estimates them jointly.

In JASP, multinomial logistic regression lives under the Generalized Linear Model interface. Go to Regression → Generalized Linear Model, set Family to Other and Model to Multinomial logistic regression, then drag the categorical outcome into Dependent Variable and your predictors into Continuous Predictors / Categorical Predictors.
