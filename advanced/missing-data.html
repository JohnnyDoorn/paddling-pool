Missing Data & Imputation

Advanced Topics

Missing data is almost inevitable in psychology research. How you handle it matters: the wrong approach can introduce bias and reduce statistical power. This page covers the types of missingness and the options available for dealing with it.

Types of missingness

Before choosing a method, you need to think about why data are missing.

Mechanism Definition Example
MCAR — Missing Completely At Random The probability of being missing is unrelated to any variable (observed or unobserved) A random equipment failure causes some data to be lost
MAR — Missing At Random The probability of being missing depends on observed variables, but not on the missing value itself Older participants more likely to skip a question about technology use, but we observe age
MNAR — Missing Not At Random The probability of being missing depends on the missing value itself (or unobserved variables) Participants with the highest depression scores are most likely to drop out of a depression study
WarningWhy this matters
  • MCAR: listwise deletion is unbiased (but wasteful)
  • MAR: listwise deletion is biased. Multiple imputation or FIML is appropriate
  • MNAR: all standard methods can produce biased results. Requires sensitivity analysis or specialised models.

In practice, MCAR is often implausible — assume MAR unless you have strong reasons otherwise.

How JASP handles missing data right now

JASP currently offers two deletion-based approaches across the standard analysis modules.

Listwise deletion (complete-case analysis) — the JASP default

Only cases with complete data on all variables involved in the analysis are included.

  • When it is acceptable: MCAR (rare in practice), or MAR with a small proportion of missingness
  • Problem: throws away information from partially complete cases; biased under MAR; reduces sample size and statistical power

Pairwise deletion — available in some analyses (e.g., correlation matrices)

Each pairwise statistic uses all cases that have valid data on the two variables involved. Different cells in a correlation matrix can therefore be based on different n.

  • Problem: can produce inconsistent estimates and occasionally yields correlation matrices that are not positive-definite — which then break downstream procedures like regression or factor analysis

What about mean substitution?

Replacing missing values with the variable mean distorts the distribution, shrinks variance, and biases correlations. Don’t do it. JASP does not offer it as a default option, and it should not be re-introduced manually.

What JASP does not currently do

Multiple Imputation (MI)

MI is the recommended approach for MAR data in modern guidance — but it is not yet available in JASP. An integration of the R package mice is in active development, but has not been released in the stable version of JASP at the time of writing. For now, MI has to be done outside JASP — in R (mice, Amelia), SPSS, or Mplus — and the pooled results reported manually.

What MI does, conceptually: it creates m complete datasets (typically 20–100) by drawing plausible values for each missing cell from a model of the observed data; each dataset is analysed separately, and estimates are combined using Rubin’s rules to give a single set of coefficients, standard errors, and p-values that correctly reflects the uncertainty introduced by missingness.

Little’s MCAR test

Not currently in JASP. Available in SPSS, or in R via naniar::mcar_test().

FIML in structural-equation models

JASP’s structural-equation modelling (SEM) module is built on lavaan, which supports Full Information Maximum Likelihood (FIML) estimation under MAR. If you are fitting an SEM (path model, latent variable model, CFA), FIML is the preferred way to handle missing data: it uses all available information without imputing values. Set the missing-data option in the SEM analysis options.

Diagnosing missingness in JASP

  1. Descriptives: the Valid column shows the number of non-missing values per variable. Compare to your total N to see where the gaps are.
  2. By group: split descriptives by a grouping variable to see whether missingness clusters in one group.
  3. Detailed patterns: for missing-value heatmaps or upset plots (which cases miss which combinations of variables), export the dataset to R and use naniar or VIM.

Reporting

TipReporting missing data

State (a) the percentage of missing data per variable, (b) the missingness mechanism you assume (MCAR/MAR/MNAR) and why, and (c) the method used to handle it.

Example: “Data were missing on [variable] for 12% of participants. Missingness appeared unrelated to observed study variables (assessed by comparing complete and incomplete cases on demographic and outcome variables), so listwise deletion was used.”

If you used multiple imputation outside JASP, also report m (the number of imputations), the imputation model (variables included), and the software used (e.g., R mice 3.x).
